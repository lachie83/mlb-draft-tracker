# Contributing

This is a personal hobby project, maintained best-effort in spare time —
there's no formal roadmap process, dedicated review team, or SLA on
responses.

## Got an idea or found a bug?
Open an issue first, describing what you'd like to do (or what's broken)
before writing code. That gives us a chance to talk it through — whether it
fits, how it should work, whether something similar is already planned —
before you invest time in a PR that might need rework.

## Pull requests
Once there's agreement on an issue, PRs are welcome. Happy to discuss the
approach or help out on the PR itself, not just review it after the fact.
Keep changes focused on what was discussed, include tests where it makes
sense (see the Testing section in the README), and expect some back-and-forth
rather than an instant merge — again, best effort, so response times will
vary.

## Reporting a security issue?
See [SECURITY.md](SECURITY.md) instead — please don't open a public issue
for vulnerabilities.
